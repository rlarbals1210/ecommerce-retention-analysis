# 이커머스 고객 이탈(Churn) · 리텐션 분석

## 문제 정의
이커머스 서비스에서 신규 고객 획득 비용은 점점 오르는 반면, 기존 고객의 이탈을 조기에
감지하고 방어하는 것은 상대적으로 저렴하고 효과적인 성장 레버다. 이 프로젝트는 다음
질문에 답하기 위해 실제 대규모 행동 로그를 분석한다.

1. 고객은 언제, 얼마나 이탈하는가? (코호트 잔존율)
2. 구매 전환 과정에서 어디서 이탈이 발생하는가? (퍼널 분석)
3. 이탈 위험 세그먼트를 어떻게 정의하고, 리텐션 캠페인의 효과를 어떻게 검증할 것인가? (A/B 테스트 설계)

핵심은 "정확도 높은 예측 모델"이 아니라, **비즈니스 문제를 정의하고 데이터로 검증 가능한
형태로 푸는 과정**을 보여주는 것이다.

## 데이터
- 출처: [REES46 Marketing Platform](https://rees46.com/) 공개 multi-category store 이벤트 로그
- 기간: 2019-10 ~ 2020-04 (월별 CSV 7개, 총 약 56GB)
- 위치: 외장 SSD (`/Volumes/SSD/Mac/이탈분석/`), 프로젝트에는 원본을 포함하지 않음 (읽기 전용)
- 스키마 (9개 컬럼):

| 컬럼 | 설명 |
|---|---|
| event_time | 이벤트 발생 시각 (UTC) |
| event_type | view / cart / remove_from_cart / purchase |
| product_id | 상품 ID |
| category_id | 카테고리 ID |
| category_code | 카테고리 코드 (결측 32.2%, 01_eda 확인) |
| brand | 브랜드 (결측 14.0%, 01_eda 확인) |
| price | 가격 |
| user_id | 사용자 ID |
| user_session | 세션 ID |

⚠️ 파일당 5~9GB로 매우 크다. 전체를 메모리에 올리지 않고, 필요한 컬럼만 선택 + dtype
최적화 + 필요 시 user_id 기준 샘플링(5~10%)으로 처리한다 (`src/load.py`).

## 방법론
- **코호트 분석**: 사용자의 첫 활동(또는 첫 구매) 월을 기준 코호트로 삼고, 경과 개월 수별
  잔존율을 히트맵으로 시각화. 여러 달 데이터가 필요하므로 파일을 이어붙이거나 샘플링해서 사용.
- **퍼널 분석**: `user_session` 단위로 view → cart → purchase 도달 여부를 플래그화하여
  단계별 전환율을 계산. (당초 `remove_from_cart` 이벤트로 장바구니 이탈을 볼 계획이었으나,
  EDA 결과 보유한 7개월 데이터 전체에 `remove_from_cart` 이벤트가 존재하지 않아 제외.
  대신 "장바구니에는 담았지만 같은 세션에서 구매로 이어지지 않은 비율"로 장바구니 이탈을
  우회 측정.)
- **이탈(churn) 정의**: 마지막 활동 이후 N일 무활동을 이탈로 정의. N은 EDA로 확인한 데이터
  기간·활동 주기를 근거로 확정하고 문서화한다 (현재 `config.py`의 `CHURN_INACTIVITY_DAYS`는
  미정 — 01_eda 완료 후 결정).
- **A/B 테스트 설계**: 실제 실험 대신, 이탈 위험 세그먼트를 대상으로 한 리텐션 캠페인의 A/B
  테스트 설계 문서(대상 정의, 지표, 표본 크기 산정, 실험 기간, 검정 방법)를 작성.

- [x] 01_eda (1~4단계): 데이터 구조·기간·이벤트 분포·결측 확인
  - 각 파일은 파일명이 가리키는 월 전체를 정확히 커버
  - 이벤트 비율(10~11월 기준): view 94.9% / cart 3.6% / purchase 1.5%
  - `remove_from_cart` 이벤트 없음 → 퍼널 설계 조정 (위 참고)
  - 결측치: `category_code` 32.2%, `brand` 14.0%
- [x] 01_eda (5단계): 유저·세션 단위 통계
  - 유저당 이벤트 수: 중앙값 5회 / 평균 20.7회 (헤비유저·봇 추정 극단치 존재)
  - 유저당 세션 수: 중앙값 2회 / 평균 4.3회, 세션당 이벤트 수: 중앙값 2회
  - 재방문 간격: 중앙값 3일 / 평균 6.5일, 75%가 8일 이내 재방문 (2달 스냅샷 기준)
  - `user_session`(UUID)은 3개 컬럼만으로도 6.6GB — 세션 관련 작업 시 category dtype 검토 필요
- [x] 02_cohort_retention: 7개월 전체, 첫 활동(view 포함)월 기준 코호트 잔존율 히트맵
  - 코호트 크기(신규 유저): 10월 302만 → 11월 370만 → 12월 458만 → 1월 439만 → 2월 423만 → 3월 411만 → 4월 451만
  - 1개월차 잔존율이 코호트가 최신일수록 하락: 10월 46% → 11월 36% → 12월 26% → 1월 23% → 2월 23% → 3월 20%
  - 가설: 11~12월 블랙프라이데이/연말 시즌 특가로 일회성 방문자가 대거 유입되며 신규 유입은
    늘었지만 재방문 의향이 낮은 유저 비중이 커져 잔존율이 하락한 것으로 추정
  - 구현 시 미래(미관측) 코호트×경과개월 조합은 0%가 아니라 NaN으로 처리해 히트맵에서 제외
    (예: 2020-04 코호트는 아직 1개월차 데이터가 없음 — 0%로 표시하면 오독 소지)
